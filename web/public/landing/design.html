How Runko is designed

Eight principles, in plain terms. Every feature either follows from one of these or doesn't get built.

01Git is the only storage.

Your code lives in a normal Git repository — no custom database of file contents, no virtual filesystem. If you ever want out, git clone is the exit, and it always works.

02The tree is the truth.

Who owns what, and which checks a project needs, are written in small files inside the repository itself. The server keeps a database, but it's only an index — you can delete it and rebuild everything from the code.

03Nobody pushes to trunk.

Every change — from a person, a bot, or an AI agent — goes through the same single gate: review, required checks, owner approval, and a secret scan. There is no side door.

04A change is an idea, not a commit.

Each change has a stable identity that survives rewriting, rebasing, and amending. You can rework the bottom of a stack and every change above it follows along — one push updates them all.

05Only re-test what could have broken.

When trunk moves while your change waits, Runko checks whether the new commits touch anything your change depends on. If they can't affect you, you land without re-running checks. When in doubt, it re-runs — it never guesses in your favor.

06Agents are users with tighter rules.

AI agents use the same commands and APIs people do, but the server holds them to stricter limits: no approving their own work, no touching paths outside their assignment, caps on how much they can change at once. Enforced on the server — never by trusting the agent.

07Start with three fields, opt into the rest.

A new project needs a name, a type, and an owner. Build files, CI wiring, and everything else are generated or added later when you actually want them — never a wall of required YAML.

08Don't rebuild what already works.

Runko never runs your CI, manages your servers, or replaces your editor. It owns change identity, ownership, and the merge gate — and mirrors your repository to GitHub, GitLab, or any Git host you already trust.
